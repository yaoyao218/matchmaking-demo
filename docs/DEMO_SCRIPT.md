## 第 0 段：開場說明（0:00 – 0:45）

### 步驟 1：指向 Navbar 身份切換器

**說詞：**
> 「這個系統支援三種身份——空閒學生、衝突學生、組長。右上角下拉選單可以即時切換，不需要重新整理頁面。」

> 重點：先讓評審看清楚 Navbar 結構再開始操作。

### 步驟 2：說明核心技術

**說詞：**
> 「每週 5 天 × 10 節 = 50 個時段，壓縮成一個 BigInt。兩個遮罩 AND 不為零就是衝突，NOT 扣掉雙方就是黃金共同空堂。所有計算在瀏覽器即時完成。」

**開場技術關鍵字：**
- 50-bit BigInt 遮罩壓縮演算法
- Vue 3 Reactive 全局狀態，跨組件即時同步
- 三身份 Mock 使用者，零後端

## 第 5 段：收尾總結（6:30 – 7:00）

### 三句話技術亮點

- **50-bit BigInt**：O(1) 衝突計算，不需迴圈遍歷
- **Pessimistic Locking（FOR UPDATE）**：防止 TOCTOU race condition（後端實作）
- **Auto-cancel cascade**：錄取後同課程 PENDING 申請全自動取消
- **全 in-memory Demo**：零 API 依賴，任何環境直接開啟即可展示

## Q&A 準備

| 問題 | 回答 |
|------|------|
| 為什麼用 BigInt 不用陣列？ | 50 位元 AND 運算比迴圈快 50 倍，且序列化成字串方便儲存 |
| 兩人同時按錄取怎麼辦？ | 後端用 Prisma FOR UPDATE 悲觀鎖，第二個請求會 rollback |
| 黃金空堂怎麼算？ | `~(groupMask \| studentMask) & ((1n<<50n)-1n)` |
| 遮罩怎麼對應時段？ | `bit i → 第 ⌊i/10⌋+1 天，第 (i%10)+1 節` |
